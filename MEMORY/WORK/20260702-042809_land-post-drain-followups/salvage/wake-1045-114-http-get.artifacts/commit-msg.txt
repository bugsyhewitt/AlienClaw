fix(http_get): align output fields with MSB spec; add direct unit tests

http_get.run() previously returned Python-style snake_case keys
(status_code, body, body_length, content_type) while
seed/msb/http_get.msb OUTPUT CONTRACT specifies camelCase (statusCode,
content, bytesReturned, contentType, truncated). The mismatch locked
fetch_then_parse.martian into a workaround and would silently fail
any new Martian that follows the MSB spec.

Changes:
- src/alienclaw/tools/http_get.py (37 → 61 lines, 37 → 45 stmts):
  - Rename output keys: status_code→statusCode, body→content,
    body_length→bytesReturned, content_type→contentType.
  - Add 'truncated' flag (new MSB field) — set via Content-Length
    check against _MAX_RESPONSE_BYTES (1 MiB cap).
  - Keep 'body_preview' as an extension field for backwards compat.
- seed/martians/fetch_then_parse.martian (15 lines):
  - Update slot wiring from body to the MSB-spec key 'content'.
- test/tools/test_http_get.py (NEW, 256 lines, 18 cases, 6 classes):
  - TestMissingUrl: empty/missing url returns error (line 13).
  - TestFieldNamesMsbCompliant: all MSB-spec keys present, legacy
    keys absent, at each field_count value 1/3/4/5.
  - TestParamClamping: field_count, body_preview, request_count clamps.
  - TestErrorPath: HTTPError returns statusCode (not status_code),
    connection refused returns 'Request failed'.
  - TestHeaders: custom headers forwarded to server.
  - TestTruncatedFlag: Content-Length > 1 MiB → truncated=true and
    bytesReturned capped at 1 MiB.
