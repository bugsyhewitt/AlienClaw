The architecture

AlienClaw runs three fixed Tier-A agents: BossBot receives goals from the user. AdvisorBot is consulted by BossBot on every non-trivial decision. CreatorBot runs campaigns — spawning ephemeral Specialists, each scoped to one campaign, each erased after it completes.

Specialists summon Martians — the actual tool workers. Every Martian is defined by a 256-character Base62 genome string. Genomes encode identity, execution parameters, behavior rules, and a checksum. The genome is the Martian's DNA.

The genome

256 characters, Base62 alphabet (0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz), four sections of 64 characters each:

  • IDENTITY (chars 0–63) — Martian ID tag, generation marker, tool family
  • EXECUTION (chars 64–127) — flow type, retry config, performance mode
  • BEHAVIOR (chars 128–191) — escalation policy, output contract
  • CHECKSUM (chars 192–255) — FNV-1a-inspired 64-char hash of sections 0–2

Mutation operates on one character at a time (rate: 1/256 per character per generation). Crossover is section-boundary: each of the three mutable sections is inherited from one parent or the other. The checksum is always recomputed after any change — produced genomes are always valid.

Cross-language compliance

The governance layer is TypeScript. The genome, brain, fitness, bridge, and evolution layers are Python. Every cross-boundary behavior — genome checksums, brain registry parsing, bridge wire protocol — is validated against shared JSON fixture files. Both language runners load the same fixture, run the same cases, and CI fails on any disagreement. This discipline has caught six bugs to date.

The environmental thesis

AI inference has a real environmental cost: electricity, cooling water, hardware manufacturing. The cost scales with compute — more tool calls, more tokens, more energy. A system that rewards efficiency is a system that rewards less harm.

AlienClaw's fitness formula rewards Martians that accomplish tasks with fewer tool calls: 

fitness = correctness × (1 / max(1, tool_calls))

A Martian that completes a task in one tool call scores 1.0. One that uses five calls scores 0.2, even if the result is equally correct. Selection pressure propagates the efficient genomes; inefficient ones die off.

The community genome network (coming with Packet 10) extends this across operators. When your AlienClaw install discovers a high-fitness genome from another operator, it adds it to your local population. The network aggregates the selection signal from thousands of runs. Over time, the globally-evolved population approaches maximum efficiency on shared tasks.

Why open source

Best case: AlienClaw gets acquired and the environmental thesis gets funded at scale. Acceptable case: open-source recognition, with the project and its ideas living on. The code and the specs are what matter. Making them freely available is how they survive.

The project is MIT-licensed. No contributor agreement. The genome codec and fitness formula are in the public domain of ideas — anyone can implement them, fork them, or build competing infrastructure that uses the same genome format. Interoperability between genome network operators is the point.
