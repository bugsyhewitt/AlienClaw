api.alienclaw.net

The AlienClaw API serves the community genome network. It accepts fitness reports from operator installs, stores high-fitness genomes, and serves the global leaderboard.

Status: deploying — api.alienclaw.net is being deployed (Packet 31). The spec is locked.

Full spec: docs/specs/LEADERBOARD_API_SPEC.md

Planned endpoints

GET /v1/genomes/top

Return the top genomes per Martian type, ranked by fitness.

GET /v1/genomes/top?martian_type=compute&limit=50

POST /v1/genomes/submit

Submit a genome + fitness score from your local population. Requires an operator token.

POST /v1/genomes/submit
Content-Type: application/json

{
  "genome": "<256-char Base62>",
  "martian_type": "compute",
  "fitness": 0.92,
  "run_count": 14,
  "operator_id": "<your-operator-token>"
}

GET /v1/leaderboard

Global leaderboard per Martian type, updated daily. This is the data source for the leaderboard page.

GET /v1/leaderboard?martian_type=compute&limit=50

GET /v1/health

API health check.

GET /v1/health
→ {"status": "ok", "version": "1.0"}

Authentication

Read endpoints (GET /v1/genomes/top, GET /v1/leaderboard) are unauthenticated — anyone can read the global leaderboard.

Write endpoints (POST /v1/genomes/submit) require an operator token sent in the Authorization: Bearer <token> header. Operator tokens will be available when the API launches.

Data format

All responses are JSON. Genomes are 256-character Base62 strings (alphabet: 0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz). Fitness scores are floats in [0.0, 1.0].

The genome format is defined in GENOME_SPEC.md. The wire protocol between the local bridge and the API is defined in LEADERBOARD_API_SPEC.md.
